## What T-2b is

T-2 built the focused transcript replica at **small scale** (full reproject + live-tail).
T-2b makes it **correct and smooth at disk scale** (thousands of items), plus two carried
Minors. Scope the exact task list in the brainstorm; the known pieces:

1. **Bounded-tail reconcile / pagination** — THE core item. Memory
   [[large-transcript-latency-spike]]: reconcile MUST be bounded-tail; full-history
   reconcile is a ~1s stall (370–3100× slower). The read frontier (`ReadRange`
   All/Delta/One + `RangeRead.watermark`, in `crates/lens-ui/src/focused/` +
   `crates/lens-core/src/persist/transcript.rs`) and the disk-render path
   ([[state-model-d23-disk-render]]: focused replica reads transcript from disk, delete
   deltas + watermark, id-upsert) are the surfaces. Also `/items` pagination on the
   client side. Virtualization is already GO on native gpui `list()`
   ([[transcript-virtualization-spike-2026-07]]).

2. **M1 (carried Minor)** — the focused `RowStore.entities` map never GCs removed rows
   (`crates/lens-ui/src/focused/rowsource.rs`). Unbounded growth over a long session.

3. **M4 (carried Minor)** — `rusqlite` (+ `bundled`) is a lens-ui dep ONLY to name
   `rusqlite::ErrorCode::DatabaseBusy|DatabaseLocked` for busy detection. Cleaner: add
   `PersistError::is_busy(&self) -> bool` to lens-core, have lens-ui call that, drop the
   dep. Wrinkle: the test fake injects busy via `PersistError::Sqlite(rusqlite::Error)`,
   so removing the dev-dep needs a lens-core rusqlite-free busy-injection helper.

4. **Demo (final acceptance task)** — per the 2026-07-22 decision, build the seeded
   focused-transcript demo LAST, on top of the finished disk-scale surface, so it shows
   the interesting behavior (a large transcript that pages + stays smooth), not a toy.
   It doubles as T-2b's visual acceptance rig (permanent, like the terminal demo rig).
   There is currently NO interactive demo of the focused transcript — `lens-app
   --features demo` shows the board only; the focused pane is empty (no reader factory
   seeded). The live path (`prepare_live` + `spawn_live_session`) needs a running omnigent.
