## Learnings (persisted to memory)
- **Real-window probe false-greens** (`[[gpui-realwindow-probe-falsegreen-traps]]`): a probe passes vacuously via a sticky latch, arming state while bounds are still 0×0 (first layout clears), or `wait_frames` shorter than the 100ms markdown reparse throttle (no real height change). `use_keyed_state` is paint-only — read entity state between frames via a handle captured during render. The RUN + a teeth-check (disable the mechanism → probe must FAIL) is the only proof.
- **Vendored + rustfmt** (`[[vendored-rustfmt-skip-marker]]`): rustfmt.toml `ignore` is nightly-only; on stable use `#![cfg_attr(rustfmt, rustfmt::skip)]` per vendored file to keep it byte-faithful to upstream while the gate's `cargo fmt --check` stays green.
- **Per-task gates must run `fmt --check`**, not just check+clippy+test — a *building* whole-branch reviewer caught the RED gate that 6 per-task no-build reviews missed.
- **cursor-delegate**: long **write-unsandboxed** composer runs died with STALLED/ERROR at the END of otherwise-successful runs (work landed on disk, commit+report did not) — recover from disk; **sandboxed** composer was stable. See `[[cursor-delegate-unsandboxed-death]]`.
